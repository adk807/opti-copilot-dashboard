The Six-Step Pipeline
Every session moves through a fixed sequence of stages. Each step's output becomes structured state consumed by the next.
1

Define Problem

User specifies the response variable (name, unit, optimisation direction) and candidate parameters (type, range or levels). A live complexity preview estimates run counts before any search begins.

Mandatory context capture — organism, class, strain, culture mode, sensitivities, and prior literature notes. This context anchors every subsequent evidence search query.

Parameter Audit

The five-tier evidence cascade searches free scientific literature APIs, scores every candidate paper, and returns one of six deterministic classifications per parameter — never a vague suggestion.

Screening

Checks whether prior screening was performed. If not, recommends a design (Full Factorial, Plackett-Burman, or DSD) sized to the parameter count. If results exist, runs full statistical diagnostics including Lack-of-Fit testing.

Recommends one of four fixed experiment scales based on iteration cycle and screening status. Scale-up beyond bench scale is hard-gated behind cycle count — this rule cannot be overridden by any scoring logic.

Scores seven DOE model families against the confirmed parameter set, screening quality, and scale. Generates the full design matrix for the winning model, or launches the sequential Bayesian loop where applicable.

SEC. 02 — DECISION ENGINE
Parameter Audit
The highest-stakes reasoning step in the app. A five-tier search cascade widens automatically when narrower evidence is thin, then a fixed scoring rubric produces one of six classifications.
TIER 1
Organism-specific — parameter + response + organism name + strain, including a parallel MeSH-indexed query
TIER 2
Genus-level — widens to genus when species-level evidence is too sparse
TIER 3
Class-level — widens further to organism class synonyms (e.g. "yeast", "gram-negative bacteria")
TIER 4
General — parameter + response only, no organism constraint, for maximum recall on niche topics
TIER 5
Citation network — depth-1 traversal from the strongest Tier 1 papers, filtered to stay on-topic
EVIDENCE SCORE — 0 TO 100
Keyword in paper title+25
Strong causal language+20
Tier 1 search space bonus+20
Published within 3 years+15
>100 citations+15
Response variable in title+15
Review / no abstract available−5 to −10
Contradictory evidence found−15
SIX-CLASS VERDICT
SUPPORTED — SPECIFIC
≥1 paper scoring ≥60 from Tier 1 or Tier 5. Include — strong organism-specific evidence.
SUPPORTED — GENERAL
≥1 paper scoring ≥60, but only from Tiers 2–4. Include with caution — evidence not organism-specific.
SUPPORTED — MECHANISTIC
≥1 paper scoring ≥60 from mechanistic decomposition only. Include — evidence exists at the mechanism level.
UNCERTAIN
Best score falls between 30 and 59. Include with a flag — recommend experimental validation.
NOT RECOMMENDED
Best score below 30, or evidence actively contradicts inclusion. Exclude or replace — a named alternative is suggested.
INSUFFICIENT EVIDENCE
Fewer than 5 papers found after the full cascade completes. Cannot classify — request an anchor reference or flag for manual review.

Hard constraint: the largest scale tier can never be recommended before the fifth optimisation cycle. This rule sits above the scoring logic entirely — it is checked first and overrides every other signal, including user override requests for earlier scale-up.
Resource Calculator
User supplies materials-per-litre and hours-per-run once. The system converts this into total grams and total hours for the recommended run count at the selected scale.
Override Path
Users may select a different permitted scale, but must acknowledge a generated consequences summary (material cost, environmental control risk) before proceeding.
SEC. 04 — DECISION ENGINE
Model Selection
Seven DOE model families are scored on a fixed numeric rubric — parameter count, type mix, screening quality, iteration cycle, and safety of extreme combinations. Highest score wins; ties break toward fewer runs.

Central Composite (CCD)

Moderate run count
Continuous parameters, moderate factor count, clear maximise/minimise goal

Box-Behnken (BBD)

Moderate run count
Continuous parameters where extreme combinations are unsafe to test

Definitive Screening (DSD)

Low run count
Larger factor counts, resource-constrained, mixed parameter types

Plackett-Burman

Very low run count
Screening not yet done, many candidate factors, first cycle only

Mixture (Simplex)

Scales with components
Parameters sum to a constant — percentages, ratios, blends

Taguchi L-array

Low–moderate run count
Categorical parameters present, robustness is the priority

Bayesian / Sequential

Small batch, adaptive
Many factors, later iteration cycles, each experiment is costly to run
All seven share one scoring pass — no model is hardcoded as a default.
